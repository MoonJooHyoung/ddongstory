# 게임 개발 개선 포인트

게임 개발자가 실제로 느끼는 개선이 필요한 사항들을 정리한 문서입니다.

---

## 1. 스탯 성장 공식 필요 ⚠️

### 현재 상태
- 스탯 존재만 정의됨 (HP, MP, ATK, DEF 등)
- 레벨업 시 어떻게 증가하는지 불명확

### 필요한 작업
- **레벨업 시 스탯 증가 공식 정의**
  - 예: `HP 증가 = 기본값 + (레벨 × 직업별 계수)`
  - 예: `ATK 증가 = 기본값 + (레벨 × 2.5)`
  
- **직업별 성장 곡선 그래프 작성**
  - 각 직업의 레벨별 스탯 변화 시각화
  - 밸런스 조정에 필수적
  - Excel/Google Sheets로 그래프 작성 권장

### 예시 공식
```gdscript
# 똥냥이 (전사) - 높은 HP 성장
hp_per_level = 15
mp_per_level = 2
atk_per_level = 3
def_per_level = 4

# 마법냥이 (마법사) - 높은 MP 성장
hp_per_level = 8
mp_per_level = 10
atk_per_level = 2
matk_per_level = 5
```

## 2. 스킬 배율 구조 세분화 ⚠️

### 현재 상태
- 스킬에 `damage` 숫자만 존재
- MMORPG에 필요한 세부 요소 부족

### 필요한 작업

#### 스킬 데이터 구조 확장
```gdscript
{
  id: "skill_001",
  name: "똥 검 휘두르기",
  job: "똥냥이",
  type: "active",
  element: "physical",
  
  # 데미지 관련
  base_damage: 100,           # 기본 데미지
  damage_multiplier: 1.2,      # 스킬 배율 (120%)
  hit_count: 1,                # 다타격 수
  damage_per_level: 5,        # 레벨당 데미지 증가
  
  # 상태이상
  status_effect: {
    type: "stun",              # 상태이상 타입
    chance: 0.1,               # 확률 (10%)
    duration: 2.0              # 지속 시간 (초)
  },
  
  # 소모량
  mp_cost_base: 10,           # 기본 MP 소모
  mp_cost_per_level: -0.5,     # 레벨당 MP 소모 감소
  mp_cost_min: 5,              # 최소 MP 소모
  
  # 쿨타임
  cooldown_base: 3.0,         # 기본 쿨타임 (초)
  cooldown_per_level: -0.1,    # 레벨당 쿨타임 감소
  cooldown_min: 1.0,           # 최소 쿨타임
  
  # 기타
  range: 1,                    # 사거리
  cast_time: 0.5,              # 시전 시간
  description: "근접 공격 스킬"
}
```

#### 구현 필요 사항
- [ ] 스킬 배율 시스템 (기본 공격력 × 배율)
- [ ] 다타격 시스템 (한 번에 여러 번 공격)
- [ ] 상태이상 시스템 (스턴, 독, 빙결 등)
- [ ] MP 소모 증가/감소 공식
- [ ] 쿨타임 레벨별 변화 시스템
- [ ] 스킬 레벨별 효과 변화

## 3. 몬스터 시스템 추가 필요 ⚠️

### 현재 상태
- 몬스터 스펙이 아직 없음
- 전투 대상이 정의되지 않음

### 필요한 작업

#### 몬스터 기본 구조
```gdscript
{
  id: "monster_001",
  name: "깨끗이 슬라임",
  level: 1,
  
  # 스탯
  max_hp: 50,
  atk: 10,
  def: 5,
  matk: 0,
  mdef: 3,
  speed: 100,
  
  # 보상
  exp: 10,                     # 경험치
  gold: 5,                     # 골드
  
  # 드랍 테이블
  drop_table: [
    {item_id: "potion_001", chance: 0.3},  # 30% 확률
    {item_id: "gold", amount: 5, chance: 1.0}
  ],
  
  # AI 패턴
  ai_type: "basic",           # basic, aggressive, defensive
  attack_range: 50,
  detection_range: 200
}
```

#### 구현 필요 사항
- [ ] 몬스터 기본 클래스 (`Monster.gd`)
- [ ] 몬스터 스탯 시스템
- [ ] 경험치 시스템
- [ ] 드랍 테이블 시스템
- [ ] 보스 몬스터 패턴 시스템
- [ ] 몬스터 AI (기본 이동, 공격, 추적)
- [ ] 몬스터 스폰 시스템

#### 보스 몬스터 특수 패턴
- [ ] 페이즈 시스템 (HP %에 따른 패턴 변화)
- [ ] 특수 스킬 패턴
- [ ] 광역 공격
- [ ] 소환 패턴

## 4. 네트워크 구조는 아직 없음 ⚠️

### 현재 상태
- 싱글 플레이어 기반 설계
- MMORPG를 위한 서버 구조 없음

### 필요한 작업

#### 단계별 접근
1. **1단계: 싱글 플레이어 완성** (현재)
   - 기본 게임플레이 완성
   - 모든 시스템 구현
   - 밸런스 조정

2. **2단계: 멀티플레이어 준비**
   - 네트워크 구조 설계
   - 서버 아키텍처 결정
     - Godot의 `ENet` 사용
     - 또는 전용 서버 (Node.js, Python 등)
   - 동기화 시스템 설계

3. **3단계: 멀티플레이어 구현**
   - 서버 구현
   - 클라이언트-서버 통신
   - 플레이어 동기화
   - 몬스터 동기화
   - 전투 동기화

#### 고려사항
- **서버 아키텍처 선택**
  - P2P vs 전용 서버
  - Godot 서버 vs 외부 서버
  
- **동기화 대상**
  - 플레이어 위치/상태
  - 몬스터 위치/상태
  - 아이템 드롭
  - 전투 결과
  
- **보안**
  - 치트 방지
  - 서버 검증
  - 데이터 암호화

### 현재는 싱글 기반 설계부터 완성
→ 나중에 멀티 붙여도 OK
