# 개발 로드맵 (실전 순서)

게임 개발을 단계별로 진행하는 실전 가이드입니다.

---

## ✅ 1단계: 개발 환경 & 기본 프로젝트 준비

### 완료된 항목
- [x] Godot 설치
- [x] 프로젝트 구조 설정
- [x] 기본 스크립트 작성
- [x] Main Scene 생성

### 다음 작업
- [ ] Godot에서 프로젝트 열기
- [ ] Main Scene 실행 테스트

**진행 속도**: 매우 빠름 ⚡

---

## ✅ 2단계: 플레이어 기본 움직임 완성 (가장 중요!) 🎮

### 구현 목표
- [x] 좌/우 이동
- [x] 점프
- [x] 중력 적용
- [x] 벽/땅 충돌 처리
- [ ] 캐릭터 애니메이션 적용

### 현재 상태
- ✅ `Player.gd` 기본 이동/점프 완성
- ✅ 충돌 박스 설정
- ✅ 테스트용 바닥 추가

### 테스트 방법
1. Godot에서 `scenes/Main.tscn` 실행
2. 방향키(←→) 또는 A/D로 이동
3. 스페이스바 또는 ↑로 점프
4. 바닥에 떨어지고 충돌하는지 확인

**이 단계 끝나면 "게임 느낌"이 바로 납니다!** 🎯

---

## 3단계: 맵(타일맵) + 충돌 시스템

### 구현 목표
- [ ] 간단한 지형 만들기
- [ ] 타일 충돌 설정
- [ ] 플레이어가 오르내릴 수 있는 플랫폼
- [ ] 맵 전환 시스템

### 작업 내용
- 타일맵 에디터로 맵 제작
- 충돌 레이어 설정
- 플랫폼 오브젝트 추가

**이게 되면 2D 사이드스크롤 형태가 완성됩니다!** 🗺️

---

## 4단계: 기본 공격 & 히트박스

### 구현 목표
- [ ] 마우스 클릭 또는 J키로 공격
- [ ] 근접 공격 판정(히트박스)
- [ ] 적이 맞으면 깜빡임 or 데미지 받음
- [ ] 공격 애니메이션

### 작업 내용
- 공격 입력 처리
- 히트박스 영역 설정
- 데미지 계산
- 피격 효과

**이 부분은 "전투 시스템의 핵심 뼈대"입니다!** ⚔️

---

## 5단계: 몬스터 1마리 추가

### 구현 목표
- [ ] 몬스터 HP 시스템
- [ ] 기본 공격력
- [ ] 간단한 AI (좌우 움직임 정도)
- [ ] 죽으면 사라짐
- [ ] 경험치 드롭

### 작업 내용
- `Monster.gd` 클래스 생성
- 몬스터 씬 생성
- AI 상태 머신
- 전투 시스템 연결

**"몬스터 잡는 게 되는 순간", 프로토타입 기능 완성!** 👹

---

## 🎯 여기까지 만들면 "게임이 된다"
